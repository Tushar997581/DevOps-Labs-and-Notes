# 📊 Linux Logs & Monitoring

Logs are one of the most important parts of Linux Administration and DevOps troubleshooting.

DevOps Engineers analyze logs to detect failures, debug applications, monitor services, and maintain production systems.

---

## 📌 What are Logs?

Logs are records of events generated by:

- Operating System
- Applications
- Services
- Security Events
- Background Processes


Logs help answer:

- Why did a service fail?
- When did an error happen?
- Who accessed the system?
- What changed recently?


---

# 📂 Important Linux Log Locations


Most Linux logs are stored inside:

```bash
/var/log
```

View log directory:

```bash
ls -l /var/log
```


Common log files:


| Log File | Purpose |
|-|-|
| /var/log/syslog | System logs |
| /var/log/auth.log | Authentication logs |
| /var/log/messages | System messages |
| /var/log/kern.log | Kernel logs |
| /var/log/nginx | Nginx logs |


---

# 🔍 Viewing Log Files


## cat Command

Display complete log:

```bash
cat /var/log/syslog
```


---

## less Command

Read large logs:

```bash
less /var/log/syslog
```


Navigation:

```text
Space → Next page

q → Quit
```


---

## tail Command


View last lines:

```bash
tail /var/log/syslog
```


Last 50 lines:

```bash
tail -50 /var/log/syslog
```


Follow live logs:

```bash
tail -f /var/log/syslog
```


Useful for real-time monitoring.


---

# 🔧 Journalctl Logs


`journalctl` is used to view systemd logs.


View all logs:

```bash
journalctl
```


---

# 📌 Service Specific Logs


Example: Nginx logs

```bash
journalctl -u nginx
```


Docker logs:

```bash
journalctl -u docker
```


SSH logs:

```bash
journalctl -u ssh
```


---

# 🔴 Live Service Monitoring


Follow real-time service logs:

```bash
journalctl -u nginx -f
```


---

# 🕒 Logs by Time


Today's logs:

```bash
journalctl --since today
```


Last 1 hour:

```bash
journalctl --since "1 hour ago"
```


Specific time:

```bash
journalctl --since "2026-07-09 10:00:00"
```


---

# ⚠️ Finding Errors


Search errors:

```bash
journalctl | grep error
```


Failed services:

```bash
systemctl --failed
```


---

# 📈 System Monitoring Commands


## CPU Monitoring

```bash
top
```


or

```bash
htop
```


Shows:

- CPU usage
- Running processes
- Memory consumption


---

# 💾 Memory Monitoring


Check RAM:

```bash
free -h
```


Example:

```text
Total   Used   Free
8GB     3GB    5GB
```


---

# 💽 Disk Monitoring


Check disk space:

```bash
df -h
```


Directory usage:

```bash
du -sh /var/log
```


---

# 🧪 Practical Troubleshooting Example


Scenario:

Nginx service is not running.


Step 1: Check status

```bash
systemctl status nginx
```


Step 2: Check logs

```bash
journalctl -u nginx
```


Step 3: Find errors

```bash
journalctl -u nginx | grep error
```


Step 4: Restart service

```bash
sudo systemctl restart nginx
```
