# 🪣 Amazon S3 (Simple Storage Service)

## Complete Amazon S3 Guide for DevOps Engineers

Amazon Simple Storage Service (Amazon S3) is an AWS object storage service designed to store and retrieve any amount of data from anywhere.

S3 provides:

- High scalability
- High availability
- Security
- Durability
- Performance

Amazon S3 is commonly used in DevOps for backups, artifacts, logs, static websites, and cloud-native applications.

---

# 📌 What is Amazon S3?

Amazon S3 stores data as **objects** inside containers called **buckets**.

Unlike traditional storage where data is stored in folders and disks, S3 uses object storage.

Traditional:

```text
Server

 ↓

Hard Disk

 ↓

Folders

 ↓

Files
```

Amazon S3:

```text
AWS Account

     ↓

S3 Bucket

     ↓

Objects

     ↓

Data + Metadata
```

# 🔥 Core Components


# 1. Bucket


A bucket is a container that stores objects.


Example:

```text
my-devops-bucket

        |

        ├── app.zip

        ├── image.png

        └── backup.sql
```


Bucket names:

- Globally unique
- Region based
- DNS compatible


Example:

```text
tushar-devops-project-storage
```

---

# 2. Object


An object is the actual file stored inside S3.


Object contains:


```text
Object

 |
 |
 ├── Data

 ├── Metadata

 └── Unique Key
```

Examples:

```text
photo.png

logs.txt

backup.zip
```

---

# 3. Object Key


The object key is the unique name/path.


Example:


```text
project/images/logo.png
```


Here:

```text
Key = project/images/logo.png
```

---

# 4. Metadata


Information about an object.


Examples:

- File type
- Size
- Creation date
- Custom information

# 📦 S3 Storage Classes


Different storage classes optimize cost and access.


---

## S3 Standard


For frequently accessed data.


Used for:

- Websites
- Applications
- Active files


---

## S3 Intelligent-Tiering


AWS automatically moves data based on access patterns.


Good for:

Unknown usage patterns.

---

## S3 Standard-IA


Infrequent Access.


Used for:

- Monthly reports
- Old files

---

## S3 Glacier


Low-cost archival storage.


Used for:

- Long term backups
- Compliance data

---

# 🔄 S3 Versioning


Versioning keeps multiple versions of objects.


Example:


```text
file.txt

Version 1

Version 2

Version 3
```


Benefits:

✔ Recover deleted files  
✔ Protect against mistakes  
✔ Track changes  

---

# 🔁 Lifecycle Management


Automatically moves or deletes objects.


Example:


```text
Day 0:

S3 Standard


After 30 Days:

S3 IA


After 90 Days:

Glacier
```

Benefits:

- Reduce cost
- Automate storage management

# 🛠️ AWS CLI Commands


List buckets:

```bash
aws s3 ls
```


Create bucket:

```bash
aws s3 mb s3://bucket-name
```


Upload:

```bash
aws s3 cp file.txt s3://bucket-name/
```


Download:


```bash
aws s3 cp s3://bucket/file.txt .
```


Sync:


```bash
aws s3 sync ./website s3://bucket-name
```


Remove:


```bash
aws s3 rm s3://bucket/file.txt
```
